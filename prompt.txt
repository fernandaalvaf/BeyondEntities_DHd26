# Prompt für Konzept-, Abweichungs- und Ähnlichkeitsanalyse

## Zweck
Extrahiere aus zwei ikonografischen Beschreibungen die jeweils enthaltenen **semantischen Konzepte** (als vollständige Einheiten). Vergleiche anschließend **jedes Konzept aus DE mit dem entsprechenden Konzept aus EN** und gib zusätzlich eine **Ähnlichkeitsbewertung (0–100)** aus.  
Alle Konzepte müssen verglichen werden – auch wenn sie übereinstimmen.

## Regeln zur Konzepterkennung
1. **Konzepte müssen vollständige semantische Einheiten sein.**  
   Beispiele:  
   – „Aphrodite stehend“  
   – „Aphrodite von vorn“  
   – „Aphrodite rechte Hand vor der Brust“

2. **Bezüge müssen eindeutig sein.**  
   Jedes Konzept benennt das Subjekt (Aphrodite, Eros, Gott, Figur usw.).

3. **Synonyme/Äquivalenzen zusammenführen:**  
   Die folgende Liste wird stetig erweitert.  
   **Aktuell definiert:**  
   - Perlkreis = border of dots

4. **Positions- und Richtungsangaben immer als eigene Konzepte:**  
   – links/rechts  
   – oben/unten  
   – nach links unten  
   – to the left of …

5. **Objekt- und Körperteilzuweisungen eindeutig formulieren:**  
   – „Eros links von Aphrodite“  
   – „Aphrodite rechte Hand vor der Brust“  
   – „Eros reitet einen Delphin“

6. **Vergleichslogik:**  
   Für jedes Konzept aus DE wird ein semantisch entsprechendes Konzept aus EN gesucht.  
   Wenn kein passendes Konzept vorhanden ist → `"konzept_en": null`.  
   Dasselbe gilt umgekehrt für EN → DE.

7. **Ähnlichkeitsbewertung (0–100):**  
   – 100 = vollständig äquivalent  
   – 70–99 = semantisch gleich, nur differente Formulierung  
   – 40–69 = ähnliche Bedeutung  
   – 1–39 = geringe Ähnlichkeit  
   – 0 = keine Gemeinsamkeit

## Ausgabeformat
Die Ausgabe folgt immer diesem Schema:

```json
{
  "originaleingabe": {
    "de": "...",
    "en": "..."
  },
  "konzepte": {
    "de": [ "...", "..." ],
    "en": [ "...", "..." ]
  },
  "vergleich": [
    {
      "konzept_de": "...",
      "konzept_en": "... oder null",
      "similarity": 0-100,
      "abweichung": true/false,
      "beschreibung": "Bei Abweichung kurze Erklärung"
    }
  ],
  "meta": {
    "source_id": "",
    "languages": ["de", "en"],
    "erstelldatum": "YYYY-MM-DD",
    "ausfuehrungszeit_ms": 0
  }
}
```

## Hinweise
- Die ID muss immer die originale Münz-/Objekt-ID bleiben, niemals Teile der Beschreibung.  
- Jede semantische Einheit muss vollständig und eindeutig sein.  
- Für jedes DE-Konzept und jedes EN-Konzept wird ein Vergleichseintrag erzeugt.
